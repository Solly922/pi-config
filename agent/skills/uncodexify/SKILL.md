# Uncodexify Product UI

Use the project's existing design system first. When none exists, favor quiet, durable product design over visual novelty.

## Avoid

- A gradient hero above every application screen.
- Excessive rounded cards, floating containers, pills, and glass effects.
- Decorative charts or metrics with no decision value.
- Repeating the same title, subtitle, icon, and card-grid composition everywhere.
- Oversized marketing headlines inside routine product workflows.
- Animation that delays actions or masks state changes.
- Invented components when a native control is clearer.

## Prefer

- Conventional page structure with strong alignment and spacing.
- Borders, dividers, whitespace, and typography before shadows.
- Existing typography; otherwise a restrained readable stack.
- Dense information where the task benefits from density.
- Real buttons for actions, links for navigation, and semantic form controls.
- Clear selected, hover, focus, loading, disabled, empty, and error states.
- Plain language and labels close to the controls they describe.

Distinctiveness should come from content, proportion, details, and interaction quality rather than ornamental effects. Accessibility and established product conventions override aesthetic preferences.
